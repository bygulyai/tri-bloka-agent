# Tri Bloka Agent

**Пост-консультационный ИИ-агент для автоматизации follow-up**

---

## Название проекта

**Tri Bloka Agent** — агент «три блока», который превращает сырые заметки с консультации в три готовых артефакта: CRM-отчёт, сообщение клиенту и PDF-презентацию.

## Проблема

После каждой консультации менеджер тратит 30-60 минут на ручную обработку: пишет заметки в CRM, сочиняет клиенту сообщение и собирает презентацию. Это повторяющаяся задача, которая:
- забирает время у продаж;
- зависит от навыка конкретного менеджера;
- приводит к ошибкам (забыли цену, перепутали тариф, не отправили follow-up).

При этом структура follow-up одинакова в любой компании: фиксируем суть встречи, пишем клиенту, показываем предложение.

## Пользователь

Менеджер по продажам, консультант или предприниматель, который проводит диагностические/продажные консультации и нуждается в автоматизации follow-up. Работает в любой сфере: курсы, консалтинг, реклама, услуги.

## Функция

Агент принимает сырые заметки с консультации и выдаёт три блока:

1. **БЛОК 1 — CRM-отчёт**: один абзац до 1000 символов с фактами встречи, прогнозом и следующим шагом.
2. **БЛОК 2 — сообщение клиенту**: структурированное письмо-продолжение разговора с привязкой к конкретным фактам консультации.
3. **БЛОК 3 — PDF-презентация**: 16-слайдовая персонализированная презентация с ценами, визуалом продукта и страницей знакомства с консультантом.

Все три блока генерируются за один запуск, данные между блоками согласованы (одни и те же имя, продукт, цены).

## Формат

- **Вход**: текстовые заметки с консультации (имя, суть, продукт, цена, следующий шаг).
- **Выход**: CRM-отчёт (текст), сообщение клиенту (текст), PDF-презентация (файл).
- **Конфигурация**: `config.json` — название компании, данные консультанта, пути, дефолтные цены. Компания заполняет один раз и использует для всех консультаций.
- **Справочники**: шаблоны каталога продуктов, маршрутизации клиентов и карты источников — компания адаптирует под свой бизнес.

## MVP

Минимально работающий продукт:

1. `config.json` с данными компании и консультанта.
2. Справочники продуктов (каталог, маршрутизация, карта источников).
3. Три навыка (skill) в Claude Code: CRM + сообщение, презентация, оркестратор.
4. PowerShell-скрипт генерации презентации (`build_presentation.ps1`).
5. Python-скрипты валидации и рендеринга PDF-страниц.
6. Примеры входа и выхода для каждого блока.

Что не входит в MVP, но возможно в следующих версиях:
- Автоматический вызов через API без Claude Code.
- Интеграция с CRM (Bitrix24, amoCRM).
- Веб-интерфейс для менеджеров.

## Технологии

| Компонент | Технология |
|-----------|-----------|
| ИИ-агент | Claude Code + CLAUDE.md |
| Навыки | Markdown skill-файлы в `references/` |
| Презентация | PowerShell + PowerPoint COM Automation |
| PDF-генерация | PowerPoint COM → PDF export |
| Валидация PDF | Python 3 + PyMuPDF (fitz) |
| Рендер страниц | Python 3 + PyMuPDF (render_pdf_pages.py) |
| Конфигурация | JSON (`config.json`) |
| Скриншот продукта | Microsoft Edge headless |
| Контроль версий | Git + GitHub |

## Быстрый старт

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/USER/tri-bloka-agent.git
cd tri-bloka-agent

# 2. Скопируйте и заполните конфигурацию
cp config.example.json config.json
# Отредактируйте config.json: название компании, данные консультанта, пути

# 3. Скопируйте и заполните справочники
cd references/
cp product-catalog.example.md product-catalog.md
cp client-routing.example.md client-routing.md
cp source-map.example.md source-map.md
# Заполните своими продуктами, маршрутизацией и источниками

# 4. Добавьте референс-презентацию и фото консультанта
# (скопируйте reference-presentation.pdf и consultant-photo.png в references/)

# 5. Установите зависимости Python
pip install PyMuPDF

# 6. Запустите Claude Code в папке проекта
claude
# и отправьте заметки с консультации
```

## Как это работает

```
Сырые заметки с консультации
        │
        ▼
  ┌─────────────────┐
  │  Tri Bloka Agent │  ← Claude Code + CLAUDE.md + навыки
  └────────┬────────┘
           │
    ┌──────┼──────┐
    ▼      ▼      ▼
 БЛОК 1  БЛОК 2  БЛОК 3
 CRM-    Клиенту  PDF-
 отчёт   письмо   презентация
```

1. Менеджер вставляет заметки в Claude Code.
2. Агент парсит факты и выбирает продукт из каталога.
3. Генерирует CRM-отчёт (БЛОК 1).
4. Пишет клиентское сообщение (БЛОК 2).
5. Собирает brief JSON и запускает `build_presentation.ps1` (БЛОК 3).
6. Валидирует PDF и удаляет временные файлы.
7. Возвращает все три блока менеджеру.

## Настройка под свою компанию

Проект полностью бизнес-агностный. Чтобы адаптировать под свою компанию:

1. **`config.json`** — название, сайт, данные консультанта, дефолтные цены, заголовки слайдов.
2. **`product-catalog.md`** — ваши продукты/услуги с описаниями, ценами, ссылками.
3. **`client-routing.md`** — логика подбора продукта под запрос клиента.
4. **`source-map.md`** — ваши скрипты продаж, базы знаний, регламенты.
5. **`reference-presentation.pdf`** — ваш шаблон презентации (слайды 5, 7-15 берутся из него).
6. **`consultant-photo.png`** — фото вашего консультанта для слайда 6.
